Messages Tab Design Notes

  • Purpose: Operator's sent message history — NOT an inbox. Shows messages sent by this operator (1:1 and campaign-generated)
  • Each row shows: Type badge (CHAT teal / CAMPANHA amber), title, body preview, recipient (name or segment tag), read status, relative time
  • "+ Nova" button: Opens ComposeMessageSheet (same sheet used from CRM customer detail)
  • Filters: Todas, Chat, Campanhas — same chip pattern as everywhere else
  • Read status: Green checkmark "Lida" vs gray circle "Não lida" — gives operator visibility into engagement

Model Design Notes

  • Customer uses String IDs (not UUID) — backend uses UUID strings as MongoDB _id, matching existing pattern
  • CustomerNote — lightweight struct, computed id from content+date since backend doesn't return a note ID
  • Campaign reuses MessageContent — same struct already defined in Message.swift (title, body, imageUrl, buttons)
  • Segment — simple model, used in pickers throughout compose/campaign sheets
  • Profile gains role field — populated from AuthResponse.role during login, used by ContentView to branch TabView
  • TimelineEntry — union type wrapping Message or CustomerNote for the customer detail timeline (sorted by date)
